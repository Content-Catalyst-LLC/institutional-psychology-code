# Cognitive Bias in Institutional Decision-Making

This advanced companion repository supports the article **Cognitive Bias in Institutional Decision-Making** in the Institutional Psychology knowledge series.

The repository provides a research-grade, synthetic-data scaffold for studying how overconfidence, anchoring pressure, confirmation pressure, conformity pressure, filtering distortion, path lock-in, metric tunnel vision, power protection, dissent capacity, corrective review, information quality, feedback openness, psychological safety, justice-sensitive voice, and institutional memory shape decision quality and bias pressure.

## Research purpose

This repository is designed for:

- synthetic-data research
- institutional bias modeling
- decision-quality simulation
- bias-pressure index construction
- filtering-distortion diagnostics
- overconfidence and anchoring analysis
- confirmation-pressure analysis
- conformity and authority-gradient review
- path-lock-in and precedent-pressure analysis
- metric tunnel-vision assessment
- power-protection and legitimacy-pressure review
- structured dissent and corrective-review analysis
- feedback-openness and psychological-safety modeling
- justice-sensitive voice and affected-community evidence review
- fragile judgment diagnostics
- high-bias environment review
- debiasing sensitivity analysis
- public governance, organizational strategy, platform governance, regulatory systems, healthcare safety, education systems, environmental governance, and technology operations examples
- reproducible multi-language workflows
- teaching and public-interest methods demonstration

It is not designed for automated decision-making about real people, employees, citizens, patients, students, applicants, agencies, firms, communities, or institutions.

## Core research question

When do institutional procedures, dashboards, expert routines, committee structures, and governance processes produce disciplined judgment — and when do they stabilize overconfidence, selective attention, groupthink, path dependence, metric blindness, power-protective interpretation, and procedural self-legitimation?

## Main variables

| Variable | Meaning |
|---|---|
| overconfidence | Tendency to overstate knowledge, underestimate uncertainty, or assume implementation capacity |
| anchoring_pressure | Dependence on initial assumptions, legacy baselines, prior forecasts, or inherited categories |
| confirmation_pressure | Selective attention to evidence that supports existing assumptions |
| conformity_pressure | Group, hierarchy, status, or consensus pressure that weakens dissent |
| filtering_distortion | Softening, reframing, delaying, or sanitizing information as it travels |
| path_lock_in | Constraint from precedent, prior investments, historical success, or institutional memory |
| metric_tunnel_vision | Overreliance on measurable proxies, dashboards, scores, or rankings |
| power_protection | Pressure to protect reputation, authority, legitimacy, budgets, or dominant narratives |
| dissent_capacity | Strength of structured dissent, red-team review, premortems, and protected challenge |
| corrective_review | Quality of audit, review, retrospective, assumption-testing, and independent challenge |
| information_quality | Relevance, accuracy, context, source diversity, and timeliness of evidence |
| feedback_openness | Capacity to hear and act on negative evidence, complaints, near misses, and weak signals |
| psychological_safety | Safety to surface uncertainty, disagreement, error, and disconfirming evidence |
| justice_voice | Inclusion of affected-community evidence, burden audits, dissenting memory, and contestability |
| institutional_bias_pressure | Composite synthetic bias-pressure score |
| decision_quality | Composite synthetic decision-quality score |
| high_resilience_decision | Binary indicator for high-quality, correction-supported judgment environments |
| fragile_judgment | High apparent decision quality with weak dissent and high filtering distortion |
| high_bias_environment | High bias pressure with weak corrective review and weak feedback openness |

## Responsible-use note

All data is synthetic. The repository is intended for conceptual modeling, public-interest research, education, and transparent methods demonstration.

Do not use this repository for:

- employee screening
- employment selection
- hiring
- promotion
- compensation
- discipline
- termination
- workplace surveillance
- individual performance management
- psychological assessment
- predictive policing
- automated eligibility determination
- coercive public administration
- ranking real communities, agencies, firms, workers, students, or institutions without context
- labeling dissenting people or marginalized communities as biased
- suppressing criticism under the language of rationality, evidence, debiasing, culture, compliance, or objectivity
- extracting testimony from affected communities without changing decisions
- legitimacy laundering for already-decided reforms, dashboards, reviews, risk frameworks, or governance agendas

Any real-world adaptation must include legal review, ethical review, qualitative evidence, historical analysis, stakeholder participation, burden audits, affected-community voice, privacy review, and attention to marginalized communities.

## Suggested workflow

1. Read `docs/research-design.md`.
2. Review `ethics/responsible-use-boundaries.md`.
3. Generate synthetic bias data with `python/generate_cognitive_bias_data.py`.
4. Run dynamic decision simulation with `python/simulate_bias_dynamics.py`.
5. Run fragile judgment diagnostics with `python/fragile_judgment_review.py`.
6. Run high-bias environment review with `python/high_bias_environment_review.py`.
7. Run debiasing sensitivity analysis with `python/debiasing_sensitivity.py`.
8. Analyze bias and decision outcomes in R with `r/cognitive_bias_modeling.R`.
9. Create SQLite schema with `sql/schema.sql`.
10. Review `validation/validation-plan.md` and `validation/checklist.md`.
11. Use `docs/github-embed-wordpress.html` for the WordPress GitHub block.
