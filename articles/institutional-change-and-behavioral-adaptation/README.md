# Institutional Change and Behavioral Adaptation

This advanced companion repository supports the article **Institutional Change and Behavioral Adaptation** in the Institutional Psychology knowledge series.

The repository provides a research-grade, synthetic-data scaffold for studying how institutions change through feedback, adaptive capacity, legitimacy, governance capacity, behavioral flexibility, path dependence, coordination quality, environmental pressure, transition burden, and distributional attention.

## Research purpose

This repository is designed for:

- synthetic-data research
- institutional change modeling
- behavioral adaptation simulation
- recursive structure-behavior feedback analysis
- legitimacy-threshold modeling
- governance-capacity diagnostics
- path-dependence and institutional drag analysis
- transition-burden review
- fragile adaptation diagnostics
- distributional attention analysis
- reproducible multi-language workflows
- teaching and public-interest methods demonstration

It is not designed for automated decision-making about real people, employees, citizens, patients, students, applicants, agencies, communities, or institutions.

## Core research question

How do institutions and behavior co-evolve over time, and under what conditions does formal change become durable, legitimate, behaviorally enacted adaptation rather than symbolic reform, behavioral lag, fragmented implementation, or burden shifting?

## Conceptual frame

The scaffold models institutional change and behavioral adaptation as a function of:

- feedback quality
- adaptive capacity
- legitimacy
- incentive alignment
- normative support
- governance capacity
- behavioral flexibility
- coordination quality
- environmental change
- path dependence
- distributional attention
- transition burden

## Main variables

| Variable | Meaning |
|---|---|
| feedback_quality | Ability to detect, transmit, and use institutional signals |
| adaptive_capacity | Capacity to revise routines under changing conditions |
| legitimacy | Perceived rightfulness, fairness, and credibility of the institution |
| incentive_alignment | Degree to which incentives support desired adaptation |
| normative_support | Degree to which norms and role expectations support change |
| governance_capacity | Capacity to interpret feedback and implement change |
| path_dependence | Strength of institutional drag and inherited constraints |
| behavioral_flexibility | Ability of actors to adjust conduct without losing coordination |
| coordination_quality | Cross-level and cross-unit alignment |
| environmental_change | External pressure requiring institutional adjustment |
| distributional_attention | Attention to who benefits and who bears transition burdens |
| transition_burden | Cost, confusion, disruption, or overload created by adaptation |
| change_score | Composite synthetic score for institutional change |
| successful_adaptation | Binary adaptation threshold indicator |
| fragile_adaptation | Adaptation that appears successful but rests on weak legitimacy |
| high_transition_burden | High transition-burden threshold indicator |

## Responsible-use note

All data is synthetic. The repository is intended for conceptual modeling, public-interest research, education, and transparent methods demonstration.

Do not use this repository for:

- employee screening
- employment selection
- hiring
- promotion
- compensation
- discipline
- termination
- workplace surveillance
- individual performance management
- psychological assessment
- predictive policing
- automated eligibility determination
- coercive public administration
- ranking real communities or institutions without context
- legitimacy laundering for already-decided reform agendas

Any real-world adaptation must include legal review, ethical review, qualitative evidence, historical analysis, stakeholder participation, and attention to marginalized communities.

## Suggested workflow

1. Read `docs/research-design.md`.
2. Review `ethics/responsible-use-boundaries.md`.
3. Generate synthetic adaptation data with `python/generate_institutional_change_data.py`.
4. Run recursive adaptation simulation with `python/simulate_recursive_institutional_change.py`.
5. Run fragile-adaptation diagnostics with `python/fragile_adaptation_review.py`.
6. Run transition-burden review with `python/transition_burden_review.py`.
7. Analyze adaptation probability in R with `r/institutional_change_modeling.R`.
8. Create SQLite schema with `sql/schema.sql`.
9. Review `validation/validation-plan.md` and `validation/checklist.md`.
10. Use `docs/github-embed-wordpress.html` for the WordPress GitHub block.
