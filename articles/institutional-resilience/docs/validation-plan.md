# Validation Plan

A real institutional resilience study should validate the model across several layers.

## 1. Construct validity

Ask whether each variable actually represents the concept it claims to measure.

Examples:

- Does a trust survey capture trust across different communities?
- Does service continuity capture equitable continuity?
- Does redundancy represent usable backup capacity or symbolic duplication?

## 2. Face validity

Review the model with practitioners, community stakeholders, institutional staff, and domain experts.

## 3. Sensitivity analysis

Test whether findings change when:

- weights are altered
- shock assumptions change
- legitimacy thresholds vary
- missing data is introduced
- outliers are removed

## 4. Distributional validation

Compare outcomes across groups, places, agencies, or communities to ensure aggregate resilience does not hide unequal burdens.

## 5. Historical validation

Compare model outputs against known institutional crisis cases, after-action reports, archives, public testimony, and documented reform histories.
