# Validation Plan

A real institutional-memory study should validate claims across several layers.

## 1. Construct validity

Confirm that indicators actually measure intended concepts.

Examples:

- Does "memory effectiveness" measure usable memory rather than storage volume?
- Does "documented retention" include context, provenance, and decision rationale?
- Does "accessibility" measure practical retrieval by relevant actors?
- Does "memory justice" include affected-community voice and histories of harm?
- Does "revisability" capture the ability to revise categories and assumptions, not only edit records?

## 2. Behavioral validation

Compare formal memory systems with actual institutional behavior:

- use of past lessons
- recurring failure
- knowledge transfer
- onboarding quality
- archive retrieval
- decision-log use
- technical documentation
- category revision
- community-memory inclusion
- key-person dependency

## 3. Institutional validation

Assess whether archives, records systems, technical systems, handoff practices, succession processes, metadata standards, and governance structures are documented.

## 4. Process-tracing validation

Ask whether the proposed sequence is plausible:

```text
experience → record → retention → retrieval → interpretation → revision → future decision
```

## 5. Sensitivity analysis

Test whether conclusions change when:

- documented retention is lower
- tacit transfer is lower
- accessibility is lower
- metadata quality is lower
- path dependence is higher
- loss fragmentation is higher
- key-person dependency is higher
- memory justice is included or excluded

## 6. Distributional validation

Check whether official memory hides unequal burden, harm, exclusion, dissent, or community knowledge.

## 7. Participatory validation

Invite affected communities, frontline staff, archivists, technical maintainers, long-tenured personnel, low-resource actors, public-interest advocates, and marginalized groups to review interpretation.
