# Method Notes

## Why model institutional memory?

Institutional memory determines whether organizations, agencies, professions, platforms, and governance systems can carry knowledge across time. Memory supports continuity, risk recognition, strategic depth, and accountability. But memory can also preserve outdated assumptions, harmful routines, path dependence, selective official narratives, and technical lock-in.

A useful model can separate:

- documented retention
- tacit knowledge transfer
- accessibility and retrieval quality
- interpretive use
- revisability
- technical continuity
- metadata quality
- distributed integration
- memory justice
- path-dependence pressure
- loss fragmentation
- selective narration
- turnover pressure
- key-person dependency

## Why synthetic data?

Synthetic data allows transparent demonstration without using confidential records, vulnerable communities, employees, students, patients, regulated entities, citizens, or sensitive administrative information.

## Why include memory justice?

Official memory is often incomplete. Institutions may preserve procedural records while forgetting histories of harm, burden, exclusion, dissent, or marginalized testimony. Memory justice asks whose memory becomes institutional memory.

## Why include path dependence?

Memory supports continuity, but continuity can become rigidity. Path dependence shows when inherited categories, routines, or success narratives narrow future possibility.

## What this scaffold cannot do

It cannot classify real organizations or communities as having good or bad memory. It cannot justify surveillance, coercion, or punitive knowledge-management systems. It cannot replace legal analysis, archival review, public participation, historical context, qualitative research, or domain expertise.
