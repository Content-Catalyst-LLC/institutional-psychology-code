# Institutional Trust and Social Stability

This advanced companion repository supports the article **Institutional Trust and Social Stability** in the Institutional Psychology knowledge series.

The repository provides a research-grade, synthetic-data scaffold for studying how consistency, competence, fairness, transparency, accountability, integrity, recognition, repair capacity, legitimacy, voluntary compliance, cooperation capacity, learning repair, arbitrariness pressure, visible violation pressure, fragmentation pressure, and administrative burden shape institutional trust and social stability.

## Research purpose

This repository is designed for:

- synthetic-data research
- institutional trust modeling
- trust and social-stability simulation
- legitimacy and voluntary-compliance analysis
- fairness and accountability diagnostics
- recognition-and-voice modeling
- repair-capacity assessment
- administrative-burden review
- visible violation and arbitrariness-pressure diagnostics
- fragile trust-environment assessment
- high-distrust pressure review
- trust-repair sensitivity analysis
- public governance, legal institutions, social welfare systems, healthcare systems, education systems, technology platforms, regulatory systems, environmental governance, and organizational trust examples
- reproducible multi-language workflows
- teaching and public-interest methods demonstration

It is not designed for automated decision-making about real people, employees, citizens, patients, students, applicants, agencies, firms, communities, or institutions.

## Core research question

When do institutional systems generate earned trust through consistent, fair, competent, accountable, and repair-capable conduct — and when do they produce brittle order, surface compliance, defensive cooperation, unequal trust, coercive stability, or distrust rooted in historical harm and institutional burden?

## Main variables

| Variable | Meaning |
|---|---|
| consistency | Predictable rule application and stable institutional conduct |
| competence | Capacity to perform core institutional functions reliably |
| fairness | Perceived procedural and distributive fairness |
| transparency | Intelligibility and visibility of institutional reasoning |
| accountability | Credible correction of errors, abuses, failures, and harms |
| integrity | Alignment between institutional claims and institutional conduct |
| recognition_voice | Whether affected people are treated as legitimate participants and knowers |
| repair_capacity | Capacity to acknowledge, correct, remember, and repair institutional harm |
| legitimacy | Perceived rightfulness, fairness, and credibility of institutional authority |
| voluntary_compliance | Degree to which compliance rests on trust and legitimacy rather than coercion |
| cooperation_capacity | Ability to coordinate across groups, roles, and institutions |
| learning_repair | Capacity to convert failure into institutional learning and redesign |
| arbitrariness_pressure | Expected arbitrariness, inconsistency, bias, or unpredictable treatment |
| visible_violation_pressure | Visible hypocrisy, corruption, abuse, failure, or institutional betrayal |
| fragmentation_pressure | Social, political, or organizational pressure that weakens collective coordination |
| administrative_burden | Proof, waiting, complexity, documentation, stigma, and procedural load |
| trust_score | Composite synthetic institutional trust score |
| social_stability | Composite synthetic social-stability score |
| high_trust | Binary threshold indicator for high-trust environments |
| high_stability | Binary threshold indicator for high-stability environments |
| fragile_trust_environment | High apparent trust with weak fairness and weak accountability |
| high_distrust_pressure | Strong arbitrariness and violation pressure with weak repair capacity |

## Responsible-use note

All data is synthetic. The repository is intended for conceptual modeling, public-interest research, education, and transparent methods demonstration.

Do not use this repository for:

- employee screening
- employment selection
- hiring
- promotion
- compensation
- discipline
- termination
- workplace surveillance
- individual performance management
- psychological assessment
- predictive policing
- automated eligibility determination
- coercive public administration
- trust scoring of real people, groups, communities, or institutions
- ranking real communities, agencies, firms, workers, students, or institutions without context
- demanding trust from marginalized communities without institutional repair
- treating distrust as irrational when it may be historically informed
- using trust-building language to suppress dissent
- replacing enforceable rights with institutional goodwill
- extracting testimony from affected communities without changing decisions
- legitimacy laundering for already-decided reforms, dashboards, reviews, risk frameworks, or governance agendas

Any real-world adaptation must include legal review, ethical review, qualitative evidence, historical analysis, stakeholder participation, burden audits, affected-community voice, privacy review, and attention to marginalized communities.

## Suggested workflow

1. Read `docs/research-design.md`.
2. Review `ethics/responsible-use-boundaries.md`.
3. Generate synthetic trust data with `python/generate_institutional_trust_data.py`.
4. Run dynamic trust simulation with `python/simulate_trust_dynamics.py`.
5. Run fragile trust diagnostics with `python/fragile_trust_environment_review.py`.
6. Run high-distrust pressure review with `python/high_distrust_pressure_review.py`.
7. Run trust-repair sensitivity analysis with `python/trust_repair_sensitivity.py`.
8. Analyze trust and stability in R with `r/institutional_trust_modeling.R`.
9. Create SQLite schema with `sql/schema.sql`.
10. Review `validation/validation-plan.md` and `validation/checklist.md`.
11. Use `docs/github-embed-wordpress.html` for the WordPress GitHub block.
