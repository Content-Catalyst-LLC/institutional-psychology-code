# Validation Plan

A real regulatory accountability study should validate claims across several layers.

## 1. Construct validity

Confirm that indicators actually measure intended concepts.

Examples:

- Does "oversight strength" measure effective oversight rather than oversight volume?
- Does "information quality" measure usable regulatory visibility?
- Does "accountability reach" measure whether powerful actors face meaningful consequence?
- Does "burden" include cost, time, documentation, legal/technical support, and psychological burden?

## 2. Behavioral validation

Compare regulation with actual behavior:

- substantive compliance
- formal compliance
- reporting behavior
- evasion patterns
- defensive documentation
- metric gaming
- correction behavior
- repeat violations

## 3. Institutional validation

Assess whether formal rules, incentives, oversight, enforcement, reporting, learning, and accountability systems are documented.

## 4. Process-tracing validation

Ask whether the proposed sequence is plausible:

```text
rule design → interpretation → compliance/evasion → oversight visibility → answerability → correction → learning
```

## 5. Sensitivity analysis

Test whether conclusions change when:

- legitimacy is lower
- information quality is lower
- capture pressure is higher
- regulatory burden is higher
- unequal accountability increases
- oversight strength increases
- enforcement becomes more or less credible
- adaptive learning is included or excluded

## 6. Distributional validation

Check whether aggregate accountability hides unequal burden, monitoring, sanction, appeal costs, or under-protection.

## 7. Participatory validation

Invite affected communities, frontline staff, regulated actors with different capacities, domain experts, public-interest advocates, and marginalized groups to review interpretation.
