# Compliance and Rule-Following Behavior

This advanced companion repository supports the article **Compliance and Rule-Following Behavior** in the Institutional Psychology knowledge series.

The repository provides a research-grade, synthetic-data scaffold for studying how legitimacy, fairness, incentives, norm support, enforcement credibility, communication quality, cognitive clarity, trust, adaptive learning, behavioral burden, selective rule application, defensive compliance, visible hypocrisy, and norm failure shape rule-following in institutional systems.

## Research purpose

This repository is designed for:

- synthetic-data research
- compliance-quality modeling
- rule-following simulation
- legitimacy and fairness diagnostics
- norm support and trust analysis
- cognitive-clarity and communication analysis
- enforcement-legitimacy interaction analysis
- clarity-burden interaction analysis
- behavioral-burden review
- selective rule-application review
- fragile compliance diagnostics
- high-burden compliance review
- defensive-compliance and compliance-theater analysis
- public administration, organizational governance, platform governance, professional ethics, and regulatory examples
- reproducible multi-language workflows
- teaching and public-interest methods demonstration

It is not designed for automated decision-making about real people, employees, citizens, patients, students, applicants, agencies, firms, communities, or institutions.

## Core research question

When does rule-following reflect legitimate, intelligible, fair, and behaviorally workable institutional order, and when does visible compliance hide coercion, burden, strategic adaptation, selective rule application, defensive documentation, or unjust obedience?

## Conceptual frame

The scaffold models compliance quality as a function of:

- legitimacy
- perceived fairness
- incentive alignment
- norm support
- enforcement credibility
- communication quality
- cognitive clarity
- trust
- adaptive learning
- behavioral burden
- selective rule application
- defensive compliance
- hypocrisy visibility
- norm failure
- contestability
- appeal capacity
- administrative burden

## Main variables

| Variable | Meaning |
|---|---|
| legitimacy | Perceived rightfulness, procedural fairness, and authority acceptance |
| fairness | Perceived fairness of rules, procedures, outcomes, and burden |
| incentive_alignment | Degree to which incentives support substantive rule purpose rather than gaming |
| norm_support | Strength of social, professional, or organizational expectations supporting rule-following |
| enforcement_credibility | Perceived consistency, proportionality, and likelihood of enforcement |
| communication_quality | Clarity, consistency, accessibility, and practical usefulness of rule communication |
| cognitive_clarity | Ease of understanding, remembering, and applying the rule under real conditions |
| trust | Confidence that the institution and other actors will behave predictably and consistently |
| adaptive_learning | Capacity to learn from compliance breakdowns and revise rules or procedures |
| compliance_burden | Learning, documentation, financial, technological, psychological, and appeal costs |
| selective_rule_application | Unequal monitoring, sanction, flexibility, or interpretation across actors |
| defensive_compliance | Compliance behavior focused on blame avoidance rather than substantive alignment |
| hypocrisy_visibility | Visibility of powerful actors avoiding rules or consequence |
| norm_failure | Breakdown of shared expectations that others are following the rule |
| compliance_quality | Composite synthetic compliance-quality score |
| high_compliance | Binary threshold indicator for high compliance quality |
| fragile_compliance | High apparent compliance with low legitimacy |
| high_burden_compliance | High apparent compliance with high burden and selective rule application |

## Responsible-use note

All data is synthetic. The repository is intended for conceptual modeling, public-interest research, education, and transparent methods demonstration.

Do not use this repository for:

- employee screening
- employment selection
- hiring
- promotion
- compensation
- discipline
- termination
- workplace surveillance
- individual performance management
- psychological assessment
- predictive policing
- automated eligibility determination
- coercive public administration
- ranking real communities, agencies, firms, workers, students, or institutions without context
- suppressing dissent under the language of compliance
- labeling inability, exclusion, ambiguity, fear, justified distrust, or principled objection as noncompliance
- legitimacy laundering for already-decided rules, compliance regimes, or governance agendas

Any real-world adaptation must include legal review, ethical review, qualitative evidence, historical analysis, stakeholder participation, burden audits, affected-community voice, and attention to marginalized communities.

## Suggested workflow

1. Read `docs/research-design.md`.
2. Review `ethics/responsible-use-boundaries.md`.
3. Generate synthetic compliance data with `python/generate_compliance_rule_following_data.py`.
4. Run dynamic rule-following simulation with `python/simulate_rule_following_dynamics.py`.
5. Run fragile compliance diagnostics with `python/fragile_compliance_review.py`.
6. Run compliance burden review with `python/compliance_burden_review.py`.
7. Run selective rule-application sensitivity with `python/selective_rule_application_sensitivity.py`.
8. Analyze compliance outcomes in R with `r/compliance_rule_following_modeling.R`.
9. Create SQLite schema with `sql/schema.sql`.
10. Review `validation/validation-plan.md` and `validation/checklist.md`.
11. Use `docs/github-embed-wordpress.html` for the WordPress GitHub block.
