# Validation Plan

A real compliance and rule-following study should validate claims across several layers.

## 1. Construct validity

Confirm that indicators actually measure intended concepts.

Examples:

- Does "compliance quality" measure substantive rule-following rather than form completion?
- Does "legitimacy" measure perceived rightfulness rather than fear?
- Does "cognitive clarity" measure real interpretability for the actors expected to comply?
- Does "burden" include learning, documentation, financial, technological, psychological, and appeal costs?

## 2. Behavioral validation

Compare formal rule design with actual behavior:

- substantive compliance
- formal compliance
- strategic compliance
- defensive documentation
- under-reporting
- over-reporting
- appeal patterns
- help requests
- rule interpretation errors

## 3. Institutional validation

Assess whether formal rules, incentives, enforcement, communication, learning, and accountability systems are documented.

## 4. Process-tracing validation

Ask whether the proposed sequence is plausible:

```text
rule design → communication → interpretation → legitimacy/trust → compliance/evasion → feedback → learning
```

## 5. Sensitivity analysis

Test whether conclusions change when:

- legitimacy is lower
- fairness is lower
- communication quality is lower
- cognitive clarity is lower
- compliance burden is higher
- selective rule application increases
- trust declines
- adaptive learning is included or excluded

## 6. Distributional validation

Check whether aggregate compliance hides unequal burden, monitoring, sanction, appeal costs, or under-protection.

## 7. Participatory validation

Invite affected communities, frontline staff, low-resource actors, monitored actors, domain experts, public-interest advocates, and marginalized groups to review interpretation.
