# Validation Plan

A real institutional-learning study should validate claims across several layers.

## 1. Construct validity

Confirm that indicators actually measure intended concepts.

Examples:

- Does "learning capacity" measure institutional revision rather than report production?
- Does "feedback quality" include timeliness, coverage, relevance, and interpretability?
- Does "memory retention" measure operationally retrievable knowledge, not just storage?
- Does "communication openness" measure safe movement of disconfirming evidence?
- Does "decision revisability" capture whether policies, incentives, metrics, and budgets can actually change?

## 2. Behavioral validation

Compare formal learning activity with actual behavior:

- feedback use
- near-miss reporting
- dissent safety
- assumption revision
- metric revision
- policy revision
- incentive revision
- memory integration
- recurrence reduction
- affected-community feedback uptake

## 3. Institutional validation

Assess whether feedback systems, memory systems, review processes, governance structures, incentives, and accountability systems are documented.

## 4. Process-tracing validation

Ask whether the proposed sequence is plausible:

```text
experience → feedback → interpretation → memory → decision revision → routine change → future behavior
```

## 5. Sensitivity analysis

Test whether conclusions change when:

- feedback quality is lower
- communication openness is lower
- psychological safety is lower
- institutional inertia is higher
- signal distortion is higher
- feedback delay is higher
- memory decay is higher
- decision revisability is included or excluded

## 6. Distributional validation

Check whether aggregate learning hides unequal voice, burden, risk, memory, or accountability.

## 7. Participatory validation

Invite affected communities, frontline staff, low-resource actors, monitored actors, domain experts, public-interest advocates, and marginalized groups to review interpretation.
