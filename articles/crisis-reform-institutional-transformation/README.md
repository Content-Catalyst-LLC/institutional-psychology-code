# Crisis, Reform, and Institutional Transformation

This advanced companion repository supports the article **Crisis, Reform, and Institutional Transformation** in the Institutional Psychology knowledge series.

The repository provides a research-grade, synthetic-data scaffold for studying how institutional crisis, legitimacy failure, feedback breakdown, coalition strength, adaptive capacity, governance alignment, power concentration, and capture risk shape reform probability and transformation depth.

## Research purpose

This repository is designed for:

- synthetic-data research
- crisis and reform modeling
- institutional transformation analysis
- reform probability estimation
- legitimacy-loss simulation
- coalition-strength scenario analysis
- governance-alignment diagnostics
- capture-risk sensitivity testing
- distributional impact review
- reproducible multi-language workflows
- teaching and public-interest methods demonstration

It is not designed for automated decision-making about real people, employees, citizens, patients, students, applicants, agencies, communities, or institutions.

## Core research question

Under what conditions does institutional crisis become meaningful reform or deep transformation rather than symbolic adjustment, elite capture, institutional drift, or authoritarian consolidation?

## Conceptual frame

The scaffold models institutional transformation as a function of:

- crisis severity
- feedback breakdown
- legitimacy failure
- adaptive capacity
- reform-window openness
- reform coalition strength
- coordination quality
- learning rate
- governance alignment
- power concentration
- capture risk
- distributional attention

## Main variables

| Variable | Meaning |
|---|---|
| crisis_severity | Magnitude and visibility of institutional disruption |
| feedback_breakdown | Degree to which warning signals, reporting channels, and learning systems fail |
| legitimacy_failure | Decline in perceived rightfulness, fairness, competence, and authority |
| adaptive_capacity | Ability to revise routines and respond under uncertainty |
| reform_window | Openness of political, organizational, and public conditions for reform |
| coalition_strength | Capacity of actors to organize around meaningful change |
| coordination_quality | Quality of cross-level and cross-unit alignment |
| learning_rate | Ability to convert crisis evidence into institutional redesign |
| governance_alignment | Fit between reform goals, authority, accountability, and implementation capacity |
| power_concentration | Distribution of authority during crisis and reform |
| capture_risk | Risk that reform is redirected toward insider, elite, or self-protective interests |
| distributional_attention | Degree to which reforms recognize unequal harm and marginalized voices |
| transformation_score | Synthetic index of reform depth and transformative potential |
| major_reform | Binary indicator for meaningful reform threshold |
| deep_transformation | Binary indicator for deeper transformation threshold |

## Responsible-use note

All data is synthetic. The repository is intended for conceptual modeling, public-interest research, education, and transparent methods demonstration.

Do not use this repository for:

- employee screening
- employment selection
- hiring
- promotion
- compensation
- discipline
- termination
- workplace surveillance
- individual performance management
- psychological assessment
- predictive policing
- automated eligibility determination
- coercive public administration
- ranking real communities or institutions without context
- legitimacy laundering for already-decided reforms

Any real-world adaptation must include legal review, ethical review, qualitative evidence, stakeholder participation, historical analysis, and attention to marginalized communities.

## Suggested workflow

1. Read `docs/research-design.md`.
2. Review `ethics/responsible-use-boundaries.md`.
3. Generate synthetic crisis data with `python/generate_crisis_reform_data.py`.
4. Run dynamic reform simulation with `python/simulate_crisis_reform_transformation.py`.
5. Analyze reform probability in R with `r/crisis_reform_modeling.R`.
6. Create SQLite schema with `sql/schema.sql`.
7. Review `validation/validation-plan.md` and `validation/checklist.md`.
8. Use `docs/github-embed-wordpress.html` for the WordPress GitHub block.
