# Validation Plan

A real path-dependence study should validate claims across several layers.

## 1. Construct validity

Confirm that indicators actually measure the intended concepts.

Examples:

- Does "switching cost" capture legal, cognitive, political, and technical transition costs?
- Does "legitimacy" reflect broad trust or only dominant-group acceptance?
- Does "complementarity" capture actual cross-system dependency?

## 2. Historical validation

Compare findings against archival sources, legislative histories, public records, administrative histories, and long-term institutional trajectories.

## 3. Process-tracing validation

Ask whether the proposed sequence is plausible:

```text
initial conditions → reinforcement → increasing returns → switching costs → lock-in
```

## 4. Sensitivity analysis

Test whether conclusions change when:

- weights are altered
- disruption pressure increases
- reform capacity increases
- legitimacy assumptions vary
- distributional burden is included or excluded
- complementarity is represented differently

## 5. Distributional validation

Check whether aggregate stability hides unequal burdens across communities.

## 6. Participatory validation

Invite affected communities, frontline staff, civil society actors, and domain experts to review interpretation.
