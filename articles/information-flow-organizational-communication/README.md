# Information Flow and Organizational Communication

This advanced companion repository supports the article **Information Flow and Organizational Communication** in the Institutional Psychology knowledge series.

The repository provides a research-grade, synthetic-data scaffold for studying how signal quality, communication quality, interpretive integration, feedback usability, memory retention, openness, escalation access, trust, community voice, digital transparency, distortion loss, overload, siloing, suppression pressure, metric tunnel vision, and false coherence shape institutional information-flow effectiveness.

## Research purpose

This repository is designed for:

- synthetic-data research
- institutional communication modeling
- information-flow simulation
- signal-quality analysis
- communication-quality diagnostics
- interpretive-integration assessment
- feedback-usability and memory-linkage analysis
- escalation-access review
- openness, trust, and psychological-safety modeling
- community-voice and communicative-accountability review
- distortion-loss and transmission-delay analysis
- overload and attention-burden review
- siloing and false-coherence diagnostics
- suppression-pressure and bad-news escalation analysis
- digital-transparency and data-provenance review
- fragile communication-system review
- high-overload communication review
- public administration, organizational strategy, platform governance, regulatory systems, healthcare safety, education systems, environmental governance, and technology operations examples
- reproducible multi-language workflows
- teaching and public-interest methods demonstration

It is not designed for automated decision-making about real people, employees, citizens, patients, students, applicants, agencies, firms, communities, or institutions.

## Core research question

When does distributed information become usable institutional knowledge that supports truthful perception, coordinated action, memory, learning, accountability, and adaptation — and when does communication produce distortion, overload, false clarity, silencing, metric tunnel vision, symbolic participation, or data-rich ignorance?

## Main variables

| Variable | Meaning |
|---|---|
| signal_quality | Accuracy, relevance, timeliness, coverage, and contextual value of generated signals |
| communication_quality | Reliability, clarity, completeness, role fit, and channel quality of information movement |
| interpretive_integration | Capacity to synthesize knowledge across roles, levels, units, data systems, and communities |
| feedback_usability | Degree to which feedback is actionable, decision-relevant, and linked to revision pathways |
| memory_retention | Degree to which information is preserved with context and available for future decisions |
| openness | Safety and willingness to surface bad news, weak signals, dissent, and uncertainty |
| escalation_access | Capacity for relevant signals to reach actors with authority to decide or revise |
| trust | Belief that communication is safe, meaningful, and institutionally consequential |
| community_voice | Inclusion and uptake of affected-community knowledge, testimony, complaints, and lived experience |
| digital_transparency | Data provenance, algorithmic traceability, category visibility, dashboard limits, and technical accountability |
| distortion_loss | Filtering, softening, reframing, delay, sanitization, or transmission loss |
| overload | Attention burden caused by excessive dashboards, meetings, alerts, reports, and messages |
| siloing | Degree to which knowledge remains trapped in units, professions, platforms, or jurisdictions |
| suppression_pressure | Incentive, fear, power, legal, reputational, or political pressure that makes truth costly |
| metric_tunnel_vision | Overreliance on measurable proxies at the expense of lived, contextual, or qualitative knowledge |
| information_effectiveness | Composite synthetic institutional information-flow effectiveness score |
| high_integration | Binary threshold indicator for strong information integration |
| fragile_communication | High apparent information effectiveness with low openness and high distortion |
| high_overload_system | High apparent information effectiveness with high overload and metric tunnel vision |

## Responsible-use note

All data is synthetic. The repository is intended for conceptual modeling, public-interest research, education, and transparent methods demonstration.

Do not use this repository for:

- employee screening
- employment selection
- hiring
- promotion
- compensation
- discipline
- termination
- workplace surveillance
- individual performance management
- psychological assessment
- predictive policing
- automated eligibility determination
- coercive public administration
- ranking real communities, agencies, firms, workers, students, or institutions without context
- suppressing dissent under the language of communication, transparency, reporting, information quality, culture, or accountability
- extracting testimony from affected communities without changing decisions
- treating silence as agreement where communication may be unsafe
- legitimacy laundering for already-decided consultation, feedback, dashboard, reporting, or governance agendas

Any real-world adaptation must include legal review, ethical review, qualitative evidence, historical analysis, stakeholder participation, burden audits, affected-community voice, communication-channel review, privacy review, and attention to marginalized communities.

## Suggested workflow

1. Read `docs/research-design.md`.
2. Review `ethics/responsible-use-boundaries.md`.
3. Generate synthetic information-flow data with `python/generate_information_flow_data.py`.
4. Run dynamic communication simulation with `python/simulate_information_flow_dynamics.py`.
5. Run fragile communication diagnostics with `python/fragile_communication_review.py`.
6. Run high-overload communication review with `python/high_overload_communication_review.py`.
7. Run distortion sensitivity analysis with `python/distortion_sensitivity.py`.
8. Analyze communication outcomes in R with `r/information_flow_modeling.R`.
9. Create SQLite schema with `sql/schema.sql`.
10. Review `validation/validation-plan.md` and `validation/checklist.md`.
11. Use `docs/github-embed-wordpress.html` for the WordPress GitHub block.
