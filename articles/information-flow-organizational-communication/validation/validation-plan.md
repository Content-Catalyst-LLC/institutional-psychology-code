# Validation Plan

A real information-flow and communication study should validate claims across several layers.

## 1. Construct validity

Confirm that indicators actually measure intended concepts.

Examples:

- Does "information effectiveness" measure usable institutional knowledge rather than communication volume?
- Does "signal quality" include context, timeliness, relevance, and source diversity?
- Does "openness" measure actual ability to surface bad news safely?
- Does "community voice" measure uptake into decisions, not just consultation?
- Does "digital transparency" include provenance, category visibility, and contestability?

## 2. Behavioral validation

Compare formal communication systems with actual communication behavior:

- upward reporting
- bad-news escalation
- dissent protection
- near-miss disclosure
- feedback follow-through
- message distortion across hierarchy
- dashboard use and misuse
- community complaint uptake
- decision traceability

## 3. Institutional validation

Assess whether reporting systems, dashboards, escalation rules, complaint channels, collaboration systems, metadata standards, and governance forums are documented.

## 4. Process-tracing validation

Ask whether the proposed sequence is plausible:

```text
signal → transmission → interpretation → escalation → memory → decision → feedback → revision
```

## 5. Sensitivity analysis

Test whether conclusions change when:

- signal quality is lower
- openness is lower
- trust is lower
- escalation access is lower
- distortion loss is higher
- overload is higher
- suppression pressure is higher
- community voice is included or excluded

## 6. Distributional validation

Check whether aggregate communication metrics hide unequal voice, burden, silence, exclusion, retaliation risk, or community knowledge.

## 7. Participatory validation

Invite affected communities, frontline staff, low-resource actors, monitored actors, domain experts, public-interest advocates, and marginalized groups to review interpretation.
