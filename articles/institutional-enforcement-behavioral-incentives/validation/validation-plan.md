# Validation Plan

A real enforcement and incentive study should validate claims across several layers.

## 1. Construct validity

Confirm that indicators actually measure intended concepts.

Examples:

- Does "monitoring quality" measure effective visibility rather than monitoring volume?
- Does "information quality" measure usable enforcement evidence?
- Does "accountability reach" measure whether decision-makers with authority face meaningful consequence?
- Does "burden" include cost, time, documentation, appeal difficulty, and psychological burden?

## 2. Behavioral validation

Compare enforcement design with actual behavior:

- substantive compliance
- formal compliance
- reporting behavior
- evasion patterns
- defensive documentation
- indicator gaming
- correction behavior
- repeat violations

## 3. Institutional validation

Assess whether formal rules, incentives, monitoring, sanctions, reporting, learning, and accountability systems are documented.

## 4. Process-tracing validation

Ask whether the proposed sequence is plausible:

```text
rule design → monitoring visibility → perceived legitimacy → compliance/evasion → sanction/correction → learning
```

## 5. Sensitivity analysis

Test whether conclusions change when:

- legitimacy is lower
- information quality is lower
- monitoring quality is higher or lower
- sanction credibility changes
- compliance burden increases
- selective enforcement increases
- accountability reach increases or decreases
- adaptive learning is included or excluded

## 6. Distributional validation

Check whether aggregate enforcement hides unequal burden, monitoring, sanction, appeal costs, or under-protection.

## 7. Participatory validation

Invite affected communities, frontline staff, monitored actors with different capacities, domain experts, public-interest advocates, and marginalized groups to review interpretation.
