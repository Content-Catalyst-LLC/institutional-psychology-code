# Decision-Making in Institutional Systems

This advanced companion repository supports the article **Decision-Making in Institutional Systems** in the Institutional Psychology knowledge series.

The repository provides a research-grade, synthetic-data scaffold for studying how bounded rationality, organizational structure, incentive alignment, information flow, legitimacy, uncertainty management, corrective capacity, justice-sensitive voice, memory quality, feedback openness, bias distortion, power-protective pressure, metric fixation, siloing, and premature closure shape institutional decision quality.

## Research purpose

This repository is designed for:

- synthetic-data research
- institutional decision-quality modeling
- bounded-rationality pressure analysis
- organizational-structure and decision-architecture review
- incentive-alignment diagnostics
- information-flow and signal-preservation analysis
- legitimacy and procedural-fairness modeling
- uncertainty-management simulation
- corrective-capacity and structured-dissent analysis
- justice-sensitive voice and affected-community evidence review
- institutional memory and feedback-openness analysis
- bias-distortion and power-protection review
- metric-fixation, siloing, and premature-closure diagnostics
- fragile decision-environment assessment
- high-distortion environment review
- public administration, regulatory governance, organizational strategy, platform governance, healthcare safety, education systems, environmental governance, and technology operations examples
- reproducible multi-language workflows
- teaching and public-interest methods demonstration

It is not designed for automated decision-making about real people, employees, citizens, patients, students, applicants, agencies, firms, communities, or institutions.

## Core research question

When do institutional procedures, hierarchies, committees, dashboards, incentives, information systems, memory structures, and governance processes produce accountable judgment — and when do they produce bounded rationality, filtered reality, symbolic review, fragile legitimacy, path dependence, or procedural confidence without institutional intelligence?

## Conceptual frame

The scaffold models institutional decision quality as a function of:

- bounded-rationality pressure

## Main variables

| Variable | Meaning |
|---|---|
| bounded_rationality_pressure | Cognitive, time, information, uncertainty, and complexity constraints on decision-makers |
| organizational_structure_quality | Quality of decision rights, escalation pathways, review structures, role clarity, and governance architecture |
| incentive_alignment | Degree to which rewarded behavior matches long-run institutional purpose |
| information_flow_effectiveness | Ability to generate, transmit, preserve, and interpret relevant information |
| legitimacy | Procedural fairness, transparency, intelligibility, contestability, and credibility of decisions |
| uncertainty_management | Capacity to use scenarios, sensitivity analysis, monitoring triggers, and staged commitments |
| corrective_capacity | Strength of review, dissent, premortems, red teams, audits, and revision mechanisms |
| justice_voice | Inclusion of affected-community knowledge, burden evidence, and marginalized voice in decision framing |
| memory_quality | Preservation of assumptions, dissent, rationale, outcomes, and lessons for future decisions |
| feedback_openness | Capacity to use negative evidence, complaints, weak signals, and outcomes to revise assumptions |
| bias_distortion | Confirmation, overconfidence, anchoring, status quo, availability, and groupthink pressures |
| power_protection | Reputational, political, legal, budgetary, or authority-protective pressure |
| metric_fixation | Overreliance on visible indicators, targets, dashboards, rankings, and proxy measures |
| siloing | Fragmentation of knowledge across units, professions, jurisdictions, or platforms |
| premature_closure | Early narrowing of alternatives, uncertainty, or dissent before adequate review |
| decision_quality | Composite synthetic institutional decision-quality score |
| high_quality_decision | Binary threshold indicator for high-quality decision environments |
| fragile_decision_environment | High apparent decision quality with weak correction and weak information flow |
| high_distortion_environment | High bias and power protection with weak feedback openness |

## Responsible-use note

All data is synthetic. The repository is intended for conceptual modeling, public-interest research, education, and transparent methods demonstration.

Do not use this repository for:

- employee screening
- employment selection
- hiring
- promotion
- compensation
- discipline
- termination
- workplace surveillance
- individual performance management
- psychological assessment
- predictive policing
- automated eligibility determination
- coercive public administration
- ranking real communities, agencies, firms, workers, students, or institutions without context
- using decision-quality language to suppress dissent
- using institutional psychology to justify opaque authority
- extracting testimony from affected communities without changing decisions
- legitimacy laundering for already-decided reforms, dashboards, reviews, risk frameworks, or governance agendas
- replacing public participation with technical scoring

Any real-world adaptation must include legal review, ethical review, qualitative evidence, historical analysis, stakeholder participation, burden audits, affected-community voice, privacy review, and attention to marginalized communities.

## Suggested workflow

1. Read `docs/research-design.md`.
2. Review `ethics/responsible-use-boundaries.md`.
3. Generate synthetic decision data with `python/generate_decision_quality_data.py`.
4. Run dynamic decision simulation with `python/simulate_decision_dynamics.py`.
5. Run fragile decision diagnostics with `python/fragile_decision_environment_review.py`.
6. Run high-distortion environment review with `python/high_distortion_environment_review.py`.
7. Run decision architecture sensitivity analysis with `python/decision_architecture_sensitivity.py`.
8. Analyze decision quality in R with `r/decision_quality_modeling.R`.
9. Create SQLite schema with `sql/schema.sql`.
10. Review `validation/validation-plan.md` and `validation/checklist.md`.
11. Use `docs/github-embed-wordpress.html` for the WordPress GitHub block.
