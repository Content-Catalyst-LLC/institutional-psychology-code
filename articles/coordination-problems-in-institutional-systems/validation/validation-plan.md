# Validation Plan

A real coordination study should validate claims across several layers.

## 1. Construct validity

Confirm that indicators actually measure intended concepts.

Examples:

- Does "communication clarity" measure message clarity or only message frequency?
- Does "authority signal" capture legitimacy as well as command strength?
- Does "high alignment" capture meaningful coordination or superficial compliance?

## 2. Behavioral validation

Compare formal coordination plans with actual behavior:

- adoption rates
- synchronized action
- timing errors
- handoff failures
- role confusion
- local workarounds
- standards fragmentation

## 3. Institutional validation

Assess whether rules, signals, focal points, communication channels, and authority structures are documented.

## 4. Process-tracing validation

Ask whether the proposed sequence is plausible:

```text
signal → shared interpretation → expectation alignment → coordinated action → feedback → signal revision
```

## 5. Sensitivity analysis

Test whether conclusions change when:

- trust is lower
- uncertainty is higher
- communication is less clear
- authority is contested
- adaptation burden increases
- competing standards are stronger
- distributional attention is included or excluded

## 6. Distributional validation

Check whether aggregate coordination hides unequal switching costs, compliance burdens, or loss of local autonomy.

## 7. Participatory validation

Invite affected communities, frontline staff, system users, domain experts, and low-resource institutions to review interpretation.
